     3‚Üí     3‚Üí## Project Status Description / Assessment
     4‚Üí     4‚Üí
     5‚Üí     5‚Üí**Project**: An interactive, premium birthday card website ("For Heena") built with Next.js 16 + App Router + TypeScript + Tailwind CSS 4 + Framer Motion + Web Audio API.
     6‚Üí     6‚Üí
     7‚Üí     7‚Üí**Current State**: Stable, fully functional, visually polished and verified end-to-end via agent-browser on both desktop (1440√ó900) and mobile (390√ó844) viewports.
     8‚Üí     8‚Üí
     9‚Üí     9‚ÜíThe site is a single-route (`/`) long-scroll experience composed of:
    10‚Üí    10‚Üí1. **IntroScreen** ‚Äî dark, code-rain canvas + glowing animated gift box + typed terminal status lines + "Open the gift" CTA.
    11‚Üí    11‚Üí2. **HeroSection** ‚Äî elegant serif "Heena" with gradient + parallax glow that follows the cursor + scroll indicator.
    12‚Üí    12‚Üí3. **MemoryDeck** ‚Äî six 3D tilt + flip memory cards (front: chapter label + glyph; back: italic quote on dark gradient).
    13‚Üí    13‚Üí4. **LoveJar** ‚Äî SVG jar with real-time liquid sloshing physics (animated path), rising bubbles, click to draw a random kept-thought.
    14‚Üí    14‚Üí5. **VinylPlayer** ‚Äî rotating metallic vinyl record + tonearm + 3 tracks with live lyrics sync + progress bar + procedural Web Audio melody.
    15‚Üí    15‚Üí6. **CakeSection** ‚Äî three candles you blow out by tap or hover-proximity; after the last flame dies, a wish message appears with a relight button.
    16‚Üí    16‚Üí7. **Footer** ‚Äî editorial divider line, signature, and a "one more sparkle" button.
    17‚Üí    17‚Üí
    18‚Üí    18‚Üí**Cross-cutting systems**:
    19‚Üí    19‚Üí- `SparkleCanvas` ‚Äî global fixed canvas overlay; any component can dispatch golden/rose/rainbow/smoke particle bursts via the `sparkle()` helper using a `heena:sparkle` CustomEvent.
    20‚Üí    20‚Üí- `BackgroundBlobs` ‚Äî fixed liquid-morphing gradient blobs + rotating ambient glow circles.
    21‚Üí    21‚Üí- `FloatingControls` ‚Äî top-right glass buttons: manual sparkle burst + ambient pad music toggle.
    22‚Üí    22‚Üí- `lib/audio.ts` ‚Äî Web Audio API engine (delay/feedback harp) with chimes, sparkle chords, ambient pad, and a procedural melody loop.
    23‚Üí    23‚Üí- `lib/birthday-data.ts` ‚Äî all content (memory cards, jar thoughts, tracks + lyrics, intro lines, wish messages, footer signature).
    24‚Üí    24‚Üí
    25‚Üí    25‚Üí**Stack confirmed**: Next.js 16.1.1, React 19, TypeScript 5, Tailwind CSS 4, Framer Motion 12, Lucide, z-ai-web-dev-sdk available.

    55‚Üí    55‚Üí
    56‚Üí    56‚Üí### Known Limitations (by design, not bugs)
    57‚Üí    57‚Üí- Audio requires a user gesture to start (browser autoplay policy). The intro "Open the gift" click + a one-time `pointerdown` listener on `window` handle this; audio will not play before the user interacts.
    58‚Üí    58‚Üí- The vinyl player uses a **procedural melody** (Web Audio synth) rather than real audio files, because no audio assets are bundled. Lyrics are timed against a mock clock that advances every 2s per chord.
    59‚Üí    59‚Üí- The intro code-rain canvas runs continuously while the intro is mounted; it is unmounted after entry, so there is no lingering cost.
    60‚Üí    60‚Üí
    61‚Üí    61‚Üí### Opportunities for the Next Phase (recommended priorities)
    62‚Üí    62‚Üí1. **Memory deck interaction polish** ‚Äî add a subtle "flip all" / "shuffle" control and a progress indicator (e.g. "2 / 6 revealed").
    63‚Üí    63‚Üí2. **Vinyl player realism** ‚Äî allow uploading a real audio file + `.lrc` lyrics (the original HTML had this); add volume + scrub controls.
    64‚Üí    64‚Üí3. **Love jar depth** ‚Äî let the user "keep" drawn thoughts into a small collection visible at the bottom; add a count of thoughts drawn.
    65‚Üí    65‚Üí4. **Cake personalization** ‚Äî let the user type a custom wish before blowing, then seal it into a card that persists across reloads (localStorage).
    66‚Üí    66‚Üí5. **Accessibility pass** ‚Äî add `aria-label`s to icon-only buttons (mostly done), keyboard focus rings, and `prefers-reduced-motion` fallbacks that disable blobs / code rain / sparkle loops.
    67‚Üí    67‚Üí6. **Theme toggle** ‚Äî add a light/dark switch via `next-themes` (dark theme tokens already exist in `globals.css`).
    68‚Üí    68‚Üí7. **Performance** ‚Äî the SparkleCanvas uses a single rAF loop with no cap on particles; cap to ~400 live particles to protect low-end devices.
    69‚Üí    69‚Üí8. **Share** ‚Äî add a "share this card" button that copies a URL (the card is a single route, so this is trivial).

   217‚Üí   217‚Üí### Bug Fixed During QA
   218‚Üí   218‚Üí- **Stale stats display**: the initial AnimatedNumber used framer-motion's `animate()` + `useInView(once:true)` gate. When stats incremented while the section was out of view, the display never updated (the `if (!inView) return` guard blocked the effect re-run on value change). Replaced with a manual rAF-based counter that animates from `fromRef.current` (previous value) to the new `value` on every change, with a brief scale-pop. Removed the `useInView` gate entirely.
   219‚Üí   219‚Üí- **Stats race condition**: the original store had a `hydrate()` function called in StatsFinale's useEffect. But MemoryDeck's `setStat` effects ran first (on mount, with empty sets ‚Üí size 0), overwriting the not-yet-hydrated store with zeros and persisting those zeros to localStorage ‚Äî wiping cumulative stats. Fixed by initializing the store synchronously from localStorage at module load (`const initialStats = load()`), eliminating the hydrate race entirely.
